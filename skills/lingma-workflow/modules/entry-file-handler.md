# Entry File Handler Module

**模块职责**: 处理入口文件的检测、创建和验证逻辑
**版本**: 1.0

---

## 核心功能

### 1. 入口检测

按优先级检测可用入口文件：

1. `.lingma/LINGMA.md`（标准入口，置信度 1.0）
2. `.lingma/workflow/README.md`（置信度 0.8）
3. `LINGMA.md` 根目录（置信度 0.7）
4. `.lingma/workflow/index.md`（置信度 0.6）

**检测逻辑**:
- 检查文件是否存在
- 验证内容质量（长度 > 100 字符 + 包含工作流相关关键词）
- 返回检测结果：`standard` | `alternative` | `missing`

### 2. 自举流程

当标准入口不存在时：

1. 扫描替代入口
2. 有替代品 → 询问用户选择（使用替代/创建新的）
3. 无替代品 → 直接启动模板创建
4. 提供三种模板：基础（~50 行）/ 完整（~200 行）/ 智能（分析项目）
5. 创建文件并记录到 `bootstrap.log`

### 3. 模板系统

- **基础模板**: 最小化配置，适合快速启动
- **完整模板**: 所有章节完整，适合正式项目
- **智能模板**: 分析项目结构后定制

模板使用 `{variable}` 占位符，运行时替换为实际值。

## 使用方式

### 在主技能中调用

读取本模块后，按以下逻辑执行：
1. 调用入口检测 → 判断入口状态
2. `standard`: 直接使用
3. `alternative`: 询问用户选择
4. `missing`: 启动自举流程

### 命令行独立使用

```bash
node bootstrap-entry.js              # 交互式引导
node bootstrap-entry.js --auto       # 自动模式
node bootstrap-entry.js --template=complete  # 指定模板
node bootstrap-entry.js --scan       # 仅扫描替代入口
```

## 验证逻辑

### 内容验证

- 长度至少 50 字符
- 包含"工作流"、"workflow"或"Lingma"关键词
- 包含标题（# 开头）和至少 2 个章节

### 质量评分

- 长度分（30 分）: >200 字符 30 分，>100 字符 20 分，>50 字符 10 分
- 关键词分（30 分）: 每个关键词 10 分，上限 30 分
- 结构分（20 分）: 每个章节 5 分，上限 20 分
- 时效分（20 分）: 7 天内 20 分，30 天内 10 分

---

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 目录无法创建 | 尝试创建父目录 |
| 文件权限不足 | 提示用户手动操作 |
| 模板加载失败 | 回退到基础模板 |
| 用户中断流程 | 保存进度，稍后继续 |
